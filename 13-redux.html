Redux

Data flow

La transmission des données entre les composants peut parfois être complexe.

Pour résoudre ce problème, Redux introduit un Store général qui sera en charge de dispatcher les données mises à jours aux composants les utilisant.

Installation

Pour pouvoir mettre en place Redux dans notre projet, nous allons récupérer 2 packages NPM :

npm install -save redux react-redux

Actions

Les Actions sont des informations envoyées au store sous forme d'objet JavaScript. Ces informations sont envoyées avec la fonction store.dispatch()

Nous allons créer un dossier "redux" contenant qui contiendra un fichier JavaScript avec l'ensemble de nos actions :

// redux/actions.js

export const USER_REGISTER = 'USER_REGISTER';
export const USER_LOGIN = 'USER_LOGIN';

export function userRegister(user) {
    return {type: USER_REGISTER, paylod: {user: user}};
}

export function userLogin(user) {
    return {type: USER_LOGIN, paylod: {user: user}};
}

Reducers

Les Reducers ont pour rôle de mettre à jour le state de l'application en fonction d'une action.

Les Reducers sont des fonctions pures prenant en entrée le state précédent et en sortie le state suivant.
